# 📊 نظام Supabase Storage والنسخ الاحتياطية

## نظرة عامة

تم دمج **نظام النسخ الاحتياطي التلقائي والتخزين السحابي** في البوت باستخدام **Supabase Storage**. الآن لديك:

✅ **نسخ احتياطي تلقائية يومية** لملفات البيانات الحساسة  
✅ **تخزين سحابي آمن** بسعة 50 ميغابايت  
✅ **أوامر إدارية** للتحكم بالنسخ الاحتياطية  
✅ **روابط عامة** للوصول إلى الملفات المرفوعة  

---

## المميزات

### 1️⃣ النسخ الاحتياطي التلقائي (Automatic Daily Backups)

**ماذا يتم نسخه احتياطياً؟**
- `users.json` - قائمة المستخدمين المسجلين في البوت
- `subscriptions.json` - بيانات الاشتراكات والإخطارات
- أي ملفات أخرى تُضيفها للنظام

**كيف تعمل؟**
- تبدأ تلقائياً عند بدء البوت
- تعمل في الخلفية دون التأثير على البوت
- تكرار يومي (كل 24 ساعة بشكل افتراضي)
- تحفظ النسخ في مجلدات منفصلة بناءً على التاريخ

**المسار في التخزين:**
```
backups/
├── daily/
│   ├── 2026/03/19/
│   │   ├── users-2026-03-19_14-30-45.json
│   │   └── subscriptions-2026-03-19_14-30-45.json
│   └── 2026/03/20/
│       └── ...
└── weekly/
    └── ...
```

### 2️⃣ تخزين الملفات المولدة (Documents Storage)

يمكنك رفع الملفات التالية مباشرة إلى التخزين:
- **الصور**: نتائج الطلاب، الرسوم البيانية
- **التقارير**: ملفات PDF و Excel
- **المستندات**: أي ملفات أخرى

الفائدة: تقليل استهلاك مساحة السيرفر المحلي + روابط دائمة

## أوامر المشرفين

### `/backup` - نسخ احتياطي يدوي فوري
```
/backup
```
**الغرض:** إنشاء نسخة احتياطية فورية لجميع الملفات المهمة  
**المخرجات:** تأكيد بأسماء الملفات المرفوعة والمسارات

---

### `/backup_status` - عرض حالة النسخ الاحتياطي
```
/backup_status
```
**الغرض:** معرفة:
- ✅ هل النسخ الاحتياطي نشط؟
- ⏱️ كل كم ساعة يتم النسخ؟
- 📅 متى تم آخر نسخ احتياطي؟

**مثال المخرجات:**
```
📊 حالة النسخ الاحتياطي:

🟢 الحالة: نشط
⏱️ الفترة الزمنية: كل 24 ساعة

📅 آخر النسخ الاحتياطية:
• users.json: 2026-03-19 14:30:45
• subscriptions.json: 2026-03-19 14:30:48
```

---

### `/upload` - رفع ملف محدد
```
/upload
```
**الغرض:** رفع ملف معين إلى التخزين  
**الخطوات:**
1. أرسل `/upload`
2. اختر الملف أو أرسل اسم الملف
3. سيتم رفعه وستتلقى رابط عام للوصول إليه

**مثال:**
```
✅ تم رفع الملف بنجاح!

📄 الملف: users.json
💾 الحجم: 45.32 KB
📍 المسار: backups/manual/2026/03/19/users-2026-03-19_14-30-45.json
🔗 الرابط: https://...
```

## كيفية الاستخدام المتقدم

### تغيير فترة النسخ الاحتياطي
في ملف `final_bot_with_image.py`، عدّل هذا السطر:

```python
# النسخ الاحتياطي كل 24 ساعة (افتراضي)
backup_scheduler = initialize_backup_scheduler(supabase, backup_interval_hours=24)

# أو غيّره إلى:
backup_scheduler = initialize_backup_scheduler(supabase, backup_interval_hours=12)  # كل 12 ساعة
backup_scheduler = initialize_backup_scheduler(supabase, backup_interval_hours=1)   # كل ساعة
```

### إضافة ملفات إضافية للنسخ الاحتياطي
في ملف `backup_scheduler.py`، عدّل داخل دالة `start()`:

```python
scheduler.start(files_to_backup=[
    "users.json",
    "subscriptions.json",
    "ملفك_الجديد.json"  # أضف ملفات إضافية
])
```

### رفع صورة أو تقرير من البوت
```python
from storage_manager import SupabaseStorageManager

manager = SupabaseStorageManager(supabase)

# رفع صورة
result = manager.upload_file("path/to/image.png", remote_folder="charts")
if result.get("success"):
    public_url = result.get("public_url")
    # ارسل الرابط للمستخدم
    bot.send_message(chat_id, f"صورتك: {public_url}")
```

## ملاحظات أمان مهمة

⚠️ **الحد الأقصى للتخزين: 50 MB**
- النسخ الاحتياطية عبارة عن ملفات JSON صغيرة جداً
- مساحة 50 MB كفيفة لسنوات من النسخ الاحتياطية
- إذا اقتربت من الحد الأقصى، ستتم إزالة النسخ القديمة تلقائياً

🔒 **الأمان:**
- البيانات مشفرة في النقل (HTTPS)
- استخدم Row Level Security في Supabase للتحكم بالوصول
- لا تشارك روابط الملفات العامة مع الجميع

---

## استكشاف الأخطاء

### المشكلة: لا يتم النسخ الاحتياطي
**الحل:**
1. تأكد من أن بيانات Supabase صحيحة في `.env`
2. تحقق من وجود الملفات المراد نسخها
3. تحقق من السجلات (logs) في الكونسول

### المشكلة: رسالة خطأ "الملف غير موجود"
**الحل:**
1. تأكد من أن الملف موجود في المجلد الرئيسي للبوت
2. استخدم المسار الكامل للملف إذا كان في مجلد فرعي

### المشكلة: الرابط العام لا يعمل
**الحل:**
1. تأكد من أن الملف رُفع بنجاح (تحقق من رسالة النجاح)
2. انتظر دقيقة واحدة قبل محاولة الوصول إلى الرابط
3. تحقق من إذن الوصول العام في Supabase

## نصائح الاستخدام الأمثل

💡 **للاستفادة القصوى من 50 MB:**

1. **احفظ روابط الملفات العامة** - لا تحتاج لإعادة رفع نفس الملف
2. **استخدم النسخ الاحتياطية للاستعادة** - في حالة حذف بيانات محلية
3. **راقب النسخ الاحتياطية** - استخدم `/backup_status` بشكل دوري
4. **حذف النسخ القديمة يدوياً** - إذا كنت بحاجة لمساحة أكثر
